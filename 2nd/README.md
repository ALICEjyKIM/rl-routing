# 다품종 SKU 거래를 위한 5PL 순차 매칭 및 가격결정 모델

본 프로젝트는 다수 화주가 보유한 다품종 SKU 재고를 구매자/수하주의 multi-SKU 주문과 연결하는 B2B 5PL 플랫폼을 모델링한다. 
핵심 아이디어는 전체 주문을 한 번에 다대다로 풀지 않고, 플랫폼에 유입된 주문을 우선순위에 따라 하나씩 처리하면서 
각 주문에 대해 여러 화주의 재고와 운송/fulfillment 옵션을 조합하는 것이다.


## 연구 문제

5PL 플랫폼은 다음 의사결정을 수행한다.

1. 유입된 구매자 주문 중 현재 처리할 주문을 선택한다.
2. 해당 주문의 SKU별 요청 수량을 여러 화주의 현재 가용 재고로 이행할 수 있는지 판단한다.
3. SKU별 최소 충족률을 만족하는 범위에서 전량 이행, 부분 이행 또는 미수락을 결정한다.
4. 매칭이 성립하면 사용된 재고와 운송 capa를 차감하고, 다음 주문은 갱신된 상태에서 처리한다.
5. 매칭 이후 구매자 지불가격과 화주 지급가격을 산정한다.

기본 모형에서는 SKU 간 대체를 허용하지 않는다. 즉, 특정 SKU가 부족하더라도 다른 SKU로 대체하여 이행하지 않는다.

## 주요 가정

- 화주는 SKU별 가용 재고, 최소 수용가격, 최소 거래비율을 플랫폼에 제시한다.
- 구매자는 복수 SKU로 구성된 주문을 제시하며, SKU별 요청 수량, 최대 지불의향, 부분 수락 허용 수준을 함께 제시한다.
- 주문의 유연 이행은 SKU별 최소 충족률 `alpha_p`로 표현한다.
- `alpha_p = 1`이면 해당 SKU는 전량 이행되어야 한다.
- `0 < alpha_p < 1`이면 최소 충족률 이상에서 부분 수락을 허용한다.
- `alpha_p = 0`이면 선택 품목으로 처리할 수 있다.
- 거래가격은 구매자의 최대 지불의향을 넘지 않고, 화주의 최소 수용가격 이상이어야 한다.
- 플랫폼은 운송 및 fulfillment 비용을 포함했을 때 거래 손실이 발생하지 않도록 가격을 결정한다.

## 매칭 모형 개요

현재 처리 중인 주문 `o`에 대해 플랫폼은 다음 요소를 고려한다.

- 화주 집합: `S`
- 주문에 포함된 SKU 집합: `P_o`
- 운송사 또는 fulfillment option 집합: `K`
- SKU별 요청 수량: `D_p`
- 화주별 SKU 재고: `I_sp`
- 구매자 최대 지불의향: `b_p`
- 화주 최소 수용가격: `a_sp`
- 운송/fulfillment 단위 비용: `c_k`
- 운송사별 처리 가능 capa: `C_k`
- 주문 예산 상한: `B_o`
- SKU 미충족 penalty: `lambda_p`

의사결정은 크게 세 가지로 요약된다.

- 주문 `o`를 수락할지 여부
- 어떤 화주-SKU-운송사 조합을 사용할지
- 각 SKU를 얼마만큼 이행하고 얼마만큼 미충족으로 둘지

목적함수는 거래 마진에서 미충족 penalty를 차감한 값을 최대화한다.

max  sum_p sum_s sum_k (b_p - a_sp - c_k) q_spk
     - sum_p lambda_p h_p

여기서 `q_spk`는 화주 `s`의 SKU `p`를 운송/fulfillment option `k`로 이행하는 수량이고, `h_p`는 SKU `p`의 미충족 수량이다.

## 핵심 제약 조건

모형은 다음 조건을 만족해야 한다.

- SKU별 총 이행 수량은 선택된 화주-SKU-운송사 조합의 공급량 합과 같다.
- 주문이 수락되면 각 SKU는 최소 충족률 이상 이행되어야 한다.
- 이행 수량은 구매자가 요청한 수량을 초과할 수 없다.
- 화주는 보유 재고 이상으로 공급할 수 없다.
- 운송/fulfillment option은 자신의 처리 가능 capa 안에서만 사용된다.
- 특정 화주의 SKU lot을 사용하면 해당 lot의 최소 거래비율 이상을 거래해야 한다.
- 구매자의 총 지불 가능 금액은 주문 예산 상한을 넘을 수 없다.

이 구조를 통해 플랫폼은 주문별로 수락 가능성, 부분 이행 수준, 재고 차감, 운송 capa 사용량을 동시에 판단한다.

## 가격결정 로직

가격결정은 매칭 이후 수행한다. 기본 방향은 다음과 같다.

- 구매자 지불가격은 SKU별 최대 지불의향 `b_p`를 초과하지 않는다.
- 화주 지급가격은 SKU별 최소 수용가격 `a_sp` 이상이어야 한다.
- 플랫폼 마진은 운송 및 fulfillment 비용을 차감한 뒤 음수가 되지 않아야 한다.
- 부분 이행 주문은 실제 이행 수량과 SKU별 충족률을 반영해 정산한다.

따라서 특정 조합 `(s, p, k)`가 거래 가능하려면 기본적으로 다음 조건이 필요하다.

b_p >= a_sp + c_k


이 조건을 만족하지 않는 조합은 가격 측면에서 거래가 성립하기 어렵다.

## 순차 처리 흐름

1. 주문 큐에서 처리할 주문 o 선택
2. 현재 재고 I_sp와 운송 capa C_k 확인
3. 주문 o에 대한 다대일 매칭 문제 해결
4. 수락 여부 및 SKU별 이행 수량 결정
5. 매칭 결과를 기준으로 가격 산정
6. 사용된 재고와 capa 차감
7. 다음 주문 처리


이 방식은 실제 플랫폼 운영처럼 주문이 시간 순서대로 들어오고, 앞선 주문의 의사결정이 이후 주문의 가능 영역을 바꾸는 상황을 반영한다.
